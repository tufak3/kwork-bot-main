# Kwork Monitor — Web Control Panel

Веб-панель для мониторинга заказов на Kwork с AI-генерацией откликов.
Работает на **GPT-OSS 120B** через Groq (быстро + дёшево + лучшее качество для русских откликов).

## Что умеет

- 🔍 Автопарсинг заказов Kwork по ключевым словам (+ исключения)
- 🤖 AI-генерация откликов через GPT-OSS 120B
- 💾 Постоянный кэш откликов — каждый заказ генерируется **только один раз**, повтор не тратит API
- 📊 Скоринг релевантности — лучшие заказы наверху, можно пропускать слабые
- ⭐ Избранное, фильтры, поиск, сортировка
- ⏰ Авто-режим: парсинг по расписанию (например, каждые 15 минут)
- ⌨️ Хоткеи: `/` — поиск, `S` — настройки, `R` — обновить

## Требования

- Node.js 18+
- Groq API ключ (бесплатный лимит — на console.groq.com)


## Настройки

Шестерёнка в правом верхнем углу — все настройки в одной модалке:

- **AI-модель** — GPT-OSS 120B (рекомендуется), 20B, или Llama
- **Авто-режим** — автопарсинг по таймеру
- **Поисковые запросы** — что искать на Kwork
- **Исключающие слова** — что пропускать (важно для экономии API)
- **AI промпт** — системный промпт (короче → дешевле)
- **Groq API ключ** — хранится в SQLite (`server/data.db`)

## Экономия API

Проект максимально экономит токены:

1. **Персистентный кэш** — каждый отклик сохраняется в SQLite. Заказ обработан → больше не оплачивается никогда.
2. **Hot-кэш в памяти** — повторные клики на ту же карточку даже SQLite не дёргают.
3. **Пре-фильтр на стороне сервера** — исключающие слова отсекают мусор до AI.
4. **Скоринг релевантности** — можно сгенерировать только для высокорелевантных заказов.
5. **Короткий ответ** — `max_tokens=600`, `reasoning_effort=low` для GPT-OSS-120B.
6. **Bulk-сохранение настроек** — одна транзакция вместо четырёх запросов.

На панели видно сколько раз ответ взят из кэша и сколько денег это сэкономило.
